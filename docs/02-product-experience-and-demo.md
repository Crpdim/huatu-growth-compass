# 产品体验与 Demo 剧本

## 1. 当前 Demo 目标

当前 Demo 使用虚构学生“林小北”：某普通本科计算机专业大二学生，成绩中游，有课程项目、无实习，希望留在省会或家乡附近。面对室友准备考研、投实习和家人建议考公，他既怕选错，也怕错过准备窗口。

产品逻辑支持考研、公考/考编和企业求职三条路径；比赛阶段使用考公内容演示完整的成长管理闭环：

```text
产品宣传首页（居中产品视频 + “开始体验”链接）
  → 进入“人生模拟器”互动体验
  → 没有人生目标：点击右上角进入原“你迷茫吗？”规划首页
  → 选择体制内人生：完成身份建档与四幕人生选择
  → 达成“模范公务员”：点击“补全任务画像”进入真实信息补充
  → 或从原规划首页继续人生课题与情境探索
  → 选择当前最想解决的人生课题
  → 8 道人生情境
  → AI 整理回答、寻找重复偏好
  → 探索画像 v0
  → 用户补充事实，并选择授权 Bilibili、GitHub、华图或 Apple Watch
  → AI 同步授权线索，补全任务画像 v1
  → AI 整理证据、匹配评分规则与检索路径资料
  → 职途初鉴：能力雷达与个人定位
  → 同路人的今天：相似案例、成长路线与生活代价
  → 三条发展路径
  → 导入用户画像到成长管理
  → 独立 Todo List 与 AI 对话
  → AI 伴学工作台与华图课程模拟检索
  → 学习反馈与能力雷达校准
  → 周期复盘和阶段结算
  → 继续当前方向 / 比较其他路径 / 生活成长计划
```

Demo 先通过产品宣传页快速说明价值，再用“先体验一种人生，再回到真实规划”的方式降低进入门槛。学生点击“开始体验”进入人生模拟器，可以直接体验体制内人生，也可以从右上角进入原规划首页；完成“模范公务员”结局后，游戏选择只作为交流起点，仍需由学生补充真实事实与授权信息，系统才会建立任务画像。学生随后参考同路人案例选择阶段性方向，进入独立成长管理页执行任务，并在伴学、反馈和阶段结算中继续修正画像与计划。

职图有声顶部只保留“初始画像、方向探索”两步。已到达节点可以点击回看。方向探索结束后，用户经过画像导入页进入职图有声的任务与伴学空间；Todo、伴学、复盘和校准不再沿用前半程步骤条。

## 2. 情绪曲线

```text
迷茫：考研、公考和实习同时出现
  → 被看见：选择很多，缺的是比较依据
  → 好奇：体制内人生为什么出现在推荐中
  → 参照：看见相似起点的人如何选择，以及需要承担什么
  → 行动：画像被导入任务管理，方向变成一组可勾选任务
  → 掌控：通过真实学习记录、完成情况和反馈持续调整计划
```

产品只承接学生主动表达的职业担忧。页面不展示焦虑指数，心理诊断和恐吓式文案均排除在体验之外。

## 3. 场景一：宣传首页与模拟人生入口

Demo 首页使用深色产品宣传页，以 30 秒演示视频为视觉中心；视频下方“开始体验”链接进入“人生模拟器”小游戏。学生可选择国企、体制内、互联网、外企、创业或网红人生；当前完整可玩分支为“体制内人生”。右上角“没有人生目标？点击探索你的人生规划 →”不会在小游戏内部展开另一套内容，而是进入原“你迷茫吗？”规划首页。

体制内分支先填写游戏人物身份，再经历舍友夜谈、入职第一周、岗位试探和十年后聚会，最终达成“模范公务员”。结局页的“补全任务画像”直接进入“让 AI 多了解你一点”页面。游戏答案不等同于真实画像，真实建模仍从用户主动填写与授权开始。

## 4. 场景二：“你迷茫吗？”

保留当前首屏：考研群、实习 Offer 和家人建议考公同时出现。

三张悬浮消息卡与鼠标形成轻量视差：不同卡片以不同深度移动，中心罗盘和轨道产生小幅错层，悬停时卡片抬起。交互只负责强化“三种未来同时靠近”的感受；触屏设备保持静态布局，系统开启“减少动态效果”时关闭视差和漂浮动画。

入口先让学生选择当前最想解决的“人生课题”，例如生活稳定、经济独立、成长上升、自主选择或“还说不清，只想先走一步”。人生课题为后续判断提供目的锚点，职业方向会在信息补充后推演。

随后进入 8 道大学生活情境，覆盖：

- 面对毕业选择时先了解什么；
- 对稳定、收入、专业成长和自主性的排序；
- 对规则型、成果型和研究型任务的偏好；
- 计划中断后的反应；
- 地域、家庭和时间约束；
- 当前最担心的问题。

每个回答绑定一条可解释线索。人生课题标注为用户主动选择，情境线索标注具体回答来源与置信度；用户可以确认、不认可或返回重选。答题结束后先播放约 2.5 秒的轻量分析动画，展示“整理回答—寻找重复偏好—形成第一印象”。这一阶段只生成临时成长标签，不评价专业能力。小游戏随后生成探索画像 `v0`，职业判断留到事实信息确认之后。

结果页使用“这是 AI 对你的第一印象”说明页面用途。左侧解释临时成长标签，右侧邀请学生逐条判断“说得像我 / 不太像我”，并把“补充资料与授权”按钮放在判断列表下方。成长罗盘全程只保留两步主进度。

示例结果：

> **人间清醒的务实派**：接下来需要看清不同生活的收益、代价与第一步。

## 5. 场景三：事实补充与授权

用户补充学校、专业、年级、成绩、做过的事情、对未来生活的想法，以及健康与家庭条件；简历为可选项，Demo 不实际读取或上传文件。健康只询问日常状态、任务强度需求或明确限制，不采集诊断信息；家庭只记录支持情况、地域限制和可投入时间等会影响规划实施的条件。页面不直接使用内部模型术语，而是通过具体问题收集选择依据。

页面增加四个可选授权入口，用户可以逐项勾选：

- Bilibili：收藏与学习分区，用于识别持续关注的学习主题；
- GitHub：公开仓库与贡献记录，用于识别项目实践和技术积累；
- 华图：课程与练习记录，用于识别已学内容、薄弱模块与学习进度；
- Apple Watch：睡眠、活动与心率趋势，只用于安排任务强度，不进行健康或心理诊断。

比赛 Demo 使用预置数据模拟同步。界面同时显示读取范围和提取出的线索，未勾选的数据源不会出现在分析依据中。

数据规则：

- 明确说明用途；
- 非必要信息可以跳过；
- 用户主动确认授权；
- 平台授权可单独开启或取消，只读取界面声明的范围；
- 探索画像不自动进入长期档案；
- 正式初始画像保存为只读基线 `v1`；
- 后续更新生成新版本，不覆盖历史。

## 6. 场景四：任务画像与三条发展路径

资料确认后，系统先展示一次可解释的 AI 分析过程：读取手动资料，同步用户勾选的平台数据，整理人生课题和情境回答，匹配八维建模规则，检索三类职业路径要求，最后检查证据冲突、信息缺口和置信度。比赛 Demo 使用本地预置数据模拟同步过程，不调用真实平台接口。

分析完成后，系统生成“八维成长画像 + 任务画像”。八项指标统一进入 0—5 分的八轴雷达，每项都展示短标签、用户可理解的结论、证据来源和置信度。健康情况用于调整任务节奏与岗位适配，家庭情况用于调整路径优先顺序；两者的分数都不是能力或个人价值评价。任务画像额外记录学习主题、项目实践、课程进度和可投入时间，用于决定任务类型、难度和复盘频率。

为了让画像在常见笔记本屏幕上更容易阅读，页面展示完整八轴雷达和八个可点选指标，学生点选任一项后查看对应结论、证据与置信度。步骤切换会回到页面顶部；资料补充页的画像快照在桌面端随页面保持可见，矮屏自动减少非必要留白。

| 维度 | 示例分数 / 状态 | 短标签 | 一句话结论 | 当前依据 |
|---|---:|---|---|---|
| 性格适配 | 4.5 | 稳健有序 | 重视规则、确定性和责任边界 | 稳定、规则与现实取舍情境 |
| 专业能力 | 2.4 | 基础起步 | 需要补充岗位筛选、行测与申论体验 | 简历预置信息与项目经历 |
| 兴趣匹配度 | 3.4 | 稳定导向 | 稳定、地域和生活节奏符合当前期待 | 人生课题与情境选择 |
| 学习与知识 | 3.1 | 基础达标 | 本科专业基础处于中游，具备继续学习的起点 | 学校、专业和成绩信息 |
| 抗压与情绪 | 4.8 | 韧性突出 | 计划受阻后会缩小任务并继续推进 | 计划中断、风险比较与规则任务情境 |
| 沟通协作 | 3.9 | 沟通稳妥 | 愿意说明顾虑并协调现实期待 | 家庭期待协商情境与项目经历 |
| 健康情况 | 4.1 | 状态稳定 | 日常状态能够支持常规学习与岗位体验 | 健康确认与日常状态记录 |
| 家庭情况 | 3.6 | 支持较强 | 家庭支持探索，但异地选择仍需提前沟通 | 家庭支持、地域偏好与可投入时间 |

综合结果使用“考公方向好像更匹配”承接八维画像，主要依据为性格适配、抗压与情绪、沟通协作，以及稳定和地域偏好。该结论只是阶段性路径线索，不等于考试成功率。“抗压与情绪”只记录情境行为；Apple Watch 样例只影响任务强度和健康条件核对，不进行心理或健康诊断。

### 同路人的今天

能力雷达之后增加真实案例参照。系统依据已确认的学校层次、应届身份、城市偏好和生活诉求，展示部分起点或诉求相似的案例，不计算虚假的精确相似度。每个案例包含：

- 双方相似的已知条件；
- 从起点到上岸和入职的关键路线；
- 当前收入、住房、通勤和工作节奏；
- 这条路线带来的收益与需要承担的代价；
- 公开来源和“个人自述”提示。

考公方向在 Demo 中提供机会样本、日常样本和风险样本各一个，防止只展示成功叙事。考研与企业求职保留可切换入口，并明确标注“样本库建设中”。用户可以从案例进入成长管理，也可以返回三条路径继续比较。

案例详情提供“问问这条路”。比赛 Demo 自动演示用户慢速输入问题、Agent 判断信息缺口、反问关键取舍、用户按钮作答、查询案例与规则资料，再结合回答形成最终建议。用户明确表达的偏好会追加为对话线索，并在进入成长管理时一起导入；线索不会直接改变职业方向。真实 AI 不可用时使用本地预置问答。

### 三条路径入口

页面同时展示：

1. 公考 / 考编；
2. 企业求职；
3. 继续升学。

每张卡包含推荐依据和信息缺口。页面使用“优先管理、作为对照、需要补充”表达当前顺序，匹配度和成功率不进入展示。

当前虚构样例中，体制内人生被标记为“本次优先验证”，依据包括：

- 用户主动选择的人生课题；
- 希望留在省会或家乡附近；
- 重视生活的可预期性；
- 愿意尝试规则明确的信息任务。

页面采用“一条重点推荐 + 两条紧凑备选”的结构。考公集中展示匹配线索、信息缺口和“进入成长管理”入口；企业求职与继续升学作为对照，避免被误解为同等优先级。

## 7. 场景五：导入画像并进入成长管理

原有“体验我的考公人生”模拟器被移除。用户在案例页或路径页点击“完成方向探索，进入成长管理”后，先看到独立应用切换页：

1. 读取画像 `v1` 与授权范围；
2. 把画像转换为任务偏好和时间约束；
3. 生成 Todo List 与复盘节点；
4. 自动进入独立行动验证页面。

切换页同时展示“职图有声 · 方向探索”和“职图有声 · 任务伴学”，再展示用户、已授权来源、本次管理方向和每一步状态。导入结束后继续使用“职图有声”品牌，前两步进度条消失；用户可通过“返回方向探索”回到前半程。

成长管理以 Agent 会话工作区为主入口，不恢复顶部功能跳转栏，也不展示周回顾和阶段总结的独立页面。左侧除“方向与本周计划”“华图课程答疑”“当前任务进展”和“本周回顾”四个会话外，增加一个明确的“任务管理”入口。任务管理采用紧凑 Todo 布局，学生可以新增、编辑、删除、完成、取消完成、暂停和恢复任务；完成状态、任务内容、AI 拆解结果和保存的资料入口都与 Agent 对话共用同一份数据。每个任务旁有独立 AI 按钮，弹窗只处理当前任务，输入框上方提供可点击的快捷建议，同时保留自由输入；演示支持把“体验 20 分钟华图入门课”拆成三个小步骤，以及为“确认 1 个可报岗位”核验并保存国家公务员局报名专题。AI 的建议必须由学生点击应用，不直接替学生改动任务。四个会话分别保存输入和回复，切换页面或会话不会清空主对话。

“方向与本周计划”采用严格的一问一答演示：学生先问“我的能力怎样”，Agent 查找已确认资料并返回包含健康、家庭在内的八轴成长画像与证据。学生再问“帮我安排一下任务看看我适不适合考公”，Agent 才生成四项低成本验证任务。学生确认税务局意向后，Agent 说明调整原因并把岗位、材料和课程任务改为税务局上下文；随后以 `checkpoint_alert` 主动提示“国考报名窗口还有 7 天”。学生回复“我已经报了名”后，Agent 记录完成事实、取消报名提醒，并把下一步重排为资格审查材料和笔试体验。每轮只新增一组消息，避免首屏一次铺满全部卡片。

当任务未开始时，AI 先询问“卡在哪里”，再根据用户选择缩小任务、顺延截止时间或保留其他路径。一次阻碍只作为对话线索，不自动写成人格结论。

进度按难度加权；拆分任务支持部分计权：

> 整体进度 = 已完成任务权重总和 ÷ 全部任务权重总和

例如学习任务权重 30 被拆成课程 10 和练习 20，只完成课程时显示 `10 / 30`，整体进度增加 10，但父任务仍在未完成清单。未完成任务按优先级、截止时间、依赖关系和权重排序；逾期项不自动删除，允许学生重新定时间或放弃并保留记录。

行动验证使用独立的 Agent 会话工作区，不与画像或案例页面混排。后续华图课程、学习反馈、关键节点提醒和周期复盘都从不同会话继续展开。

## 8. 场景六：周复盘与动态校准

Demo 将时间推进到周日晚，用户在“本周回顾”会话主动提问。Agent 直接说明已经完成与仍待验证的事项：用户愿意了解具体岗位，但还没有形成真实的学习体验。

下一阶段计划包含三项调整：

- 将完整模考降级为 20 分钟华图入门体验；
- 保留考公方向，同时继续保留企业求职；
- 将下次检测从月底提前到 3 天后，及时获取真实感受。

页面同时展示已完成、未完成、未完成原因和画像变化。点击“基于当前进度，开启下一阶段”后，系统依次演示汇总完成情况、调整任务难度和生成新计划，最终得到一份三天验证计划：体验华图考公入门课、记录愿意承担的代价、与企业求职路径做一次对照。成功状态明确展示总投入、复盘时间和“计划进行中”，完整跑通“安排—阻碍—调整—进度—复盘—重新规划”闭环。

## 9. 六分钟演示脚本

1. **0:00—0:45**：从“人生模拟器”选择体制内人生，说明右上角可进入原规划首页；
2. **0:45—1:35**：快速完成四幕选择，达成“模范公务员”，点击“补全任务画像”；
3. **1:35—2:15**：补充虚构事实，勾选 Bilibili、GitHub、华图等授权来源，展示 AI 同步线索；
4. **2:15—2:50**：展示八维成长画像、任务画像和三条发展路径；
5. **2:50—3:30**：查看同路人机会、生活和风险案例，并向案例 Agent 追问；
6. **3:30—3:45**：点击“进入成长管理”，展示“正在导入用户画像”过渡动画；
7. **3:45—4:40**：进入 Agent 主对话，依次提问能力与考公适配，展示八维画像和四项验证任务；
8. **4:40—5:25**：确认税务局意向，触发报名七天提醒，再回复已报名并查看任务重排；
9. **5:25—6:00**：切换到“当前任务进展”和“本周回顾”会话，展示 Agent 如何汇总完成状态、解释缺口并建议下一步。

完整演示可以继续在课程答疑会话中模拟检索华图课程、完成学习任务并更新任务证据；首版不再跳转独立复盘或阶段结算页。

## 10. Demo 成功标准

- 首页用人生体验降低规划门槛，游戏结局和原规划首页都能进入真实画像补全；
- 人生课题作为目的锚点，职业方向结合更多信息推演；
- 画像线索可确认、驳回或返回重选；
- 八维画像口径清晰：八项指标都进入八轴雷达，每项均展示结论、来源和置信度，界面不暴露内部权重或提示词；
- 三条路径同时存在，体制内只是当前完整演示分支；
- 推荐明确展示依据和信息缺口；
- 同路人案例同时展示机会、日常生活和真实代价；
- 授权数据源可逐项开关，并明确读取范围与用途；
- 从方向页进入行动页时明确展示画像导入过程；
- Agent 对话是成长管理主入口，结构化消息直观展示依据、缺口、行动和确认状态；
- 任务可以在计划或进展会话中直接勾选，也可以通过输入框管理；
- 状态异常直接进入当前对话，不使用打断式弹窗；
- 一般情绪不进入长期画像；
- 体验反馈先作为候选证据等待确认；
- 路径页面不展示成功率，结果文案不承诺上岸；
- 任务未完成时先询问阻碍，再调整计划，不将单次行为直接解释为人格；
- 进度按任务难度加权，计划调整展示触发原因和前后变化；
- 最终落到短期可执行、可验收的验证任务；
- 华图资源只在具体任务中出现；
- 学习任务结果能够形成可撤回的成长证据，并驱动能力雷达小幅更新；
- 阶段结束时展示完成状态、原因、行为总结和画像变化；
- 学生可以继续当前方向、比较其他路径或选择生活成长计划；
- 所有关键页面使用静态数据，现场无需依赖 AI 接口。
